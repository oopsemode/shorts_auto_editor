# Security Policy

## Supported Versions

Security fixes are handled on the latest public version of the project.

## Reporting A Vulnerability

Please report suspected vulnerabilities through GitHub Security Advisories when available. If advisories are not enabled, open a private maintainer contact channel instead of posting exploit details publicly.

Include:

- A clear description of the issue.
- Steps to reproduce.
- Impact and affected files or commands.
- Whether the issue can expose local files, execute commands, or leak private media.

## Sensitive Data

Do not include real videos, real subtitles, generated MP4 files, personal paths, or private filenames in issues, pull requests, logs, screenshots, or examples.

## External Tools

MP4 generation uses FFmpeg. Only run commands against files you trust, and review command arguments before processing private media.
